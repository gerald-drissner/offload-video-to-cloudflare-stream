# Contributing

Before submitting changes:

1. Run `php -l` on all PHP files.
2. Keep all user input sanitized and all output escaped.
3. Use WordPress APIs for HTTP requests, options, transients, cron, capabilities, and post meta.
4. Do not add third-party libraries unless their license is GPL-compatible.
5. Keep the public WordPress.org release in sync with `readme.txt` and the main plugin header.
